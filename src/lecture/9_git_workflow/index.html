Виды аутентификации

  • HTTPS
  • SSH

HTTPS аутентификация

При использовании такого типа аутентификации нужно будет при выполнении команд, связанных с удалённым репозиторием, всегда вводить свой логин и пароль.

Дополнительные телодвижения не требуются.

SSH аутентификация

В случае же с SSH аутентификацией нужны криптографические ключи.

Необходимо создать (если нет) ключи, и загрузить открытый ключ на GitHub (или другой сервис хостинга репозиториев гита).

С этим подходом, для начала работы нужно выполнить много всяких операций, но потом, во время работы, вообще не нужно будет никак участвовать в процессе аутентификации, т.е. каждый раз вводить ничего не потребуется.

Работа с удалённым репозиторием

Добавление удалённого репозитория

                        
git remote add имя адрес_репозитория

git remote add origin git@github.com:Tgjmjgj/WpfTestGit.git

Список удалённых репозиториев

                        
git remote -v

Связывание локальных и удалённых ветвей

                        
git branch -u имя_удалённого_репозитория/имя_ветви

git branch -u origin/master

Отправка изменений

push

                        
git push
git push имя_удалённого_репозитория имя_ветви

git push origin master

Получение изменений

fetch

Команда fetch используется для получения данных из удалённого репозитория. 

                        
git fetch

Fetch не выполняет слияния полученной ветки с локальной, он просто сохраняет её отдельно, под именем имя_удалённого_репозитория/имя_ветви (origin/master).

Это бывает нужно для ручных слияний.

pull

Команда pull получает данные из удалённого репозитория и выполняет с ней автоматическое слияние соответствующей локальной ветви.

pull = fetch + merge 

                        
git pull

Обработка конфликтов

Ручное слияние с помощью mergetool.

Нужно установить какую-нибудь утилиту для выполнения конфликтных слияний.

Например, KDiff3, или можно использовать расширения для IDE (Git for Visual Studio).

Настройка mergetool

                        
git config --global --add merge.tool kdiff3
git config --global --add mergetool.kdiff3.path "C:/Program Files/KDiff3/kdiff3.exe"
git config --global --add mergetool.kdiff3.trustExitCode false

git config --global --add diff.guitool kdiff3
git config --global --add difftool.kdiff3.path "C:/Program Files/KDiff3/kdiff3.exe"
git config --global --add difftool.kdiff3.trustExitCode false

Использование mergetool

                        
git mergetool
